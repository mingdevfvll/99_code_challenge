# REVIEW PROTOCOL — Problem 2: Fancy Form

> QA checklist for AI Agents performing code review on this project.
> This document enforces gate-based verification: each phase must be fully cleared
> and confirmed by the human before proceeding to the next phase.
>
> Reference documents: OVERVIEW.md · UI_BRIEF.md · TASKS.md
>
> Principle: verify intent → verify structure → verify logic → verify UI → verify quality.
> Never skip a gate. Never self-approve. Wait for explicit human confirmation.

---

## How to Use This Document

1. Work through each **Phase** sequentially
2. For every checklist item: run the check, report the result (✅ pass / ❌ fail / ⚠️ warning)
3. At the end of each Phase: present a summary to the human
4. **Do not proceed to the next Phase until the human explicitly says "continue"**
5. If any ❌ is found: fix it, re-verify that item, then re-report before asking to continue
6. Document every finding — do not silently fix without reporting

### 3.3 usePrices Hook (src/hooks/usePrices.ts)
Open file and verify:
- [ ] Uses `useQuery` from `@tanstack/react-query`
- [ ] `queryKey` is `['prices']`
- [ ] `staleTime` set to `60_000` (1 minute)

- [ ] Fetch URL is exactly `https://interview.switcheo.com/prices.json`

- [ ] Tokens with `price <= 0` or missing price are filtered out
- [ ] Result sorted alphabetically by symbol
- [ ] Return shape: `{ tokens: Token[], isLoading: boolean, isError: boolean }`
- [ ] Response typed as `TokenPrice[]` — not `any[]`

**Manual logic test — run in browser console or write a unit assertion:**
```ts

  { currency: 'ETH', date: '2023-08-29T07:10:40.000Z', price: 1645.9 },

// [{ symbol: 'ETH', price: 1645.9, icon: '/images/ETH.svg' }]
```
